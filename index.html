NPM : node package manager

What is package manager ?
A package manager is a program that lets us easily install software that other people have written on our computer or in our projects , These pieces of software are called package , a package manager also lets us install specific versions of each package , so we can easily upgrade or downgrade a package , and the package manager will note which version of a package we are using in a file call package.json so that we can easily download the right versions again in the future , or so that other developer can download the right versions if we’re collaborating with others
by using the package manager to install packages written by other people , we can build our own projects much more quickly , since we don’t have to write all of the code from scratch
npm is one of the most popular package manager
npm is not the only package manager for JavaScript EcoSystem , there is another package manager available called Yarn and Pnpm

Installing the Yarn Package Manager

Installing the Yarn Package Manager

There are a few ways to install Yarn . You can, ironically enough, install it through npm :

npm install -g yarn

If you don’t have npm installed, you can also install with a simple bash script:

curl -o- -L https://yarnpkg.com/install.sh | bash

Testing Yarn Installation

Run this to see if Yarn was properly installed or to see if you have the latest version:

yarn --version

more Details ..
Composer is a tool for dependency management in PHP. It allows us to declare the libraries our project depends on and it will manage (install/update) them for us
Composer is not a package manager in the same sense as Yum or Apt are. Yes, it deals with "packages" or libraries, but it manages
them on a per-project basis, installing them in a directory inside your project. By default,

Composer does not install anything globally .
Thus, it is a dependency manager. It does however support a "global" project for convenience via the global command.
This idea is not new , and Composer is strongly inspired by node's npm and ruby's bundler .

we can telling Composer which packages our project depends on , by using the require key inside the composer.json file
require takes an object that maps package names to version constraints ... like : 

                                {
                                  "require": {
                                      "monolog/monolog": "2.0.*"
                                  }
                              }
